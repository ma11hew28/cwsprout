Class SC.FieldView

Extends: SC.Control, SC.Validatable, SC.View
Defined in: frameworks/sproutcore/views/field/field.js
Version: 1.0

Generic base class for working with views that depend on the "input" HTML tag such as text fields.

You can work with subclasses of SC.FieldView or extend this class with your own values as well. Unlike most other HTML elements, web browsers have built-in support for editing the value of an input field. This class handles blending the browser input methods with the editing and events handling provided by the framework.

Property Details

errorLabel

The human readable label for this field for use in error strings.

-   Object errorLabel

Discussion

This is either the fieldLabel or a humanized form of the fieldKey.

fieldKey

name of key this field should display as part of a form.

-   Object fieldKey

Discussion

If you add a field as part of an SC.FormView, then the form view will automatically bind the field to the property key you name here on the content object.

fieldLabel

The human readable label you want shown for errors.

-   Object fieldLabel

Discussion

May be a loc string.

If your field fails validation, then this is the name that will be shown in the error explanation.

fieldValue

The raw value of the field, ignoring validation.

-   Object fieldValue

Discussion

You generally should not override this. Instead override setFieldValue() and getFieldValue().

value

The value of the field.

-   Object value

Discussion

The form view will pick up whatever value is published here. Generally you do not need to observe this property directly. Instead you should override setFieldValue(), getFieldValue() and the error property.
